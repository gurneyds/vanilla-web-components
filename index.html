Vanilla Web Components

This site demonstrates native web components and how they might be constructed. Each section demonstrates a potential solution when working with native web components.

Sub Components

A component may be complex and need to be broken down into several sub-components. These sub-components will probably need to be fed information from the parent component. This sample demonstrates how sub-components can be children of a parent component and receive configuration information as well as operational information.

Self Contained components

Some components will be able to stand alone and have no need for interaction with other elements on the page. The tag is an example of this behavior. The component is configured via attributes with all of the information necessary for it's operation.

Proxy Web Component

It may make sense to externalize the gathering of the data outside of a component. The purpose of a proxy component would be to gather the necessary data and then pass it on to it's children.

Proxy Web Component

The data needs of the children components may not match the provided data. An event intercepter is demonstrated that can transform the original data into a form that is consumable by the children components.
